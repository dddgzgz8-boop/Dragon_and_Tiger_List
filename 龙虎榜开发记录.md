## 二、技术架构

### 数据层
- `STORE` 对象集中管理所有数据，结构：
  ```
  STORE.cycles['5-6月2026'].people → 人员基础信息
  STORE.cycles['5-6月2026'].total → Tab1 总榜数据
  STORE.cycles['5-6月2026'].profit → Tab2 利润榜
  ...以此类推
  ```
- 预留了 3-4月 / 1-2月周期数据槽位，直接填入即可切换
- 后续接 API 只需替换 `STORE.cycles` 数据源

### 渲染层
- `renderPodium()` — 动态生成领奖台 Top3 卡片
- `renderTabNav()` / `renderAllTabs()` / `renderTab(tabId)` — 动态渲染表格
- `buildTable()` / `buildRow()` — 根据 tabId 生成不同表结构
- `renderAll()` — 全量重渲染入口

### 交互功能
| 功能 | 实现 |
|---|---|
| 列排序 | 点击表头切换升序/降序，带 ▲▼ 指示器 |
| 搜索过滤 | 200ms 防抖，按姓名/店铺筛选 |
| 分页 | 超 10 条自动出现分页器，每 Tab 独立页码 |
| 手动刷新 | 🔄 按钮触发 `renderAll()` |
| 定位自己 | 切换到总榜 + 高亮当前用户 + 滚动到视野中央 |
| 周期切换 | drop-down 切换 5-6月/3-4月/1-2月 |
| 行点击 | 弹窗展示人员明细卡片 |
| 排名变动高亮 | `prevRank` 上升的行短暂闪绿光 |

## 三、配色体系

### 设计原则
中台 besko-os 是浅色主题 `#f5f6f8` + 深绿主色 `#1d6b54`。  
龙虎榜保持浅色底，引入**金 / 蓝 / 紫 / 琥珀**四个辅助色，形成多色竞技风格。

### CSS 变量总览
```css
/* 底色 & 表面 */
--bg-page: #f4f5f7;      --bg-surface: #ffffff;
--bg-elevated: #f9fafb;  --bg-header: #f1f3f6;  --bg-stripe: #fafbfc;

/* 边框 */
--border-subtle: #e8ebf0;  --border-medium: #dde1e7;

/* 文字 */
--text-primary: #1a212b;  --text-secondary: #4a5567;  --text-muted: #8792a3;

/* 多色体系 */
--green: #1d6b54;   /* 主操作 / 成功 */
--blue: #3b6ff5;    /* 信息 / 数据 / 倒计时 */
--amber: #c27a0a;   /* 强调 / 暖色徽章 */
--violet: #7c3aed;  /* 管理功能 / 装饰 */

/* 金银铜 */
--gold: #8b6914;    --gold-bright: #d4a520;
--silver: #545e70;  --silver-bright: #95a0b2;
--bronze: #7d4a28;  --bronze-bright: #b87840;

/* 阴影体系 */
--shade-xs → --shade-xl (rgba(16,24,40, 0.03→0.10))
```

### 颜色分工
| 颜色 | 用途 |

| 🟢 绿 | 按钮、Tab激活、排序图标、分页器、状态指示 |
| 🔵 蓝 | 分数列、倒计时数字、搜索框focus、定位自己高亮 |

| 🟣 紫 | 管理员按钮/面板、Tab装饰线收尾 |

## 六、后续接入说明

### 替换数据源
```js
// 当前数据结构
STORE.cycles['5-6月2026'].total = [
  { pid:'kayo', score:328, profitGrowth:12800, newProducts:8, newSkuOk:true, prevRank:1 },
  ...
]

// 接 API 时替换为：
async function loadData() {

  STORE.cycles['5-6月2026'] = await res.json();
  renderAll();
}
```
